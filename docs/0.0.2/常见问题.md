# 常见问题

## 功能相关

### Q：评估模式有什么区别？

**A**：四种预设模式的主要区别在于分析的严格程度：

- **安全模式**：最保守，容易发现更多问题，可能有误报
- **节省模式**：平衡模式，适合日常使用
- **激进模式**：最宽松，可能漏掉一些问题
- **自定义模式**：完全由您控制各项参数

建议首次使用选择「安全模式」，了解资源状况后再调整。

### Q：报告中的主机IP列为空？

**A**：可能的原因：

1. 虚拟机处于关机状态
2. 虚拟机断开连接
3. 主机未配置管理网络接口（vmk0）

这是正常情况，不影响报告的其他内容。

### Q：为什么有些虚拟机没有性能数据？

**A**：性能数据采集需要虚拟机处于开机状态：

- 关机的虚拟机无法采集性能指标
- 断开连接的虚拟机无法采集性能数据
- 模板虚拟机通常不开机，所以没有性能数据

这些虚拟机仍然会出现在报告中，但只显示配置信息。

### Q：分析结果为空怎么办？

**A**：可能的原因：

1. **采集时间太短**：某些分析需要足够的历史数据
   - 僵尸 VM 检测建议至少 7 天数据
   - 潮汐检测建议至少 14 天数据

2. **虚拟机数量太少**：某些分析需要足够的样本

3. **阈值设置不合理**：自定义模式的阈值可能过于严格

建议使用「安全模式」并延长采集周期。

### Q：报告生成失败怎么办？

**A**：检查以下事项：

1. 确保任务已完成采集和分析
2. 检查磁盘空间是否充足
3. 查看错误日志了解具体原因

## 性能相关

### Q：采集任务需要多长时间？

**A**：采集时间取决于：

- 虚拟机数量
- 采集周期
- 网络延迟

**参考**：
- 50 台虚拟机，采集 7 天指标：约 20-30 分钟
- 100 台虚拟机，采集 30 天指标：约 1-2 小时

### Q：采集会影响生产环境吗？

**A**：不会。采集操作：

- 使用只读权限访问云平台
- 仅查询基本信息和性能数据
- 不执行任何修改操作
- 性能影响可忽略不计

### Q：可以同时运行多个任务吗？

**A**：可以。每个任务的数据是隔离的，互不影响。

## 数据相关

### Q：旧数据会被覆盖吗？

**A**：不会。每次采集都会创建新的任务，历史数据保留。

如需清理旧数据，可以：
- 删除特定任务
- 清空所有数据（在设置中）

### Q：数据存储在哪里？

**A**：数据存储在本地应用目录中：

- **Windows**：`%APPDATA%\justfit`
- **macOS**：`~/Library/Application Support/justfit`
- **Linux**：`~/.local/share/justfit`

可以通过设置环境变量 `JUSTFIT_DATA_DIR` 自定义数据目录。

### Q：如何备份数据？

**A**：备份数据目录即可：

```bash
# 备份整个数据目录
cp -r ~/.local/share/justfit ~/backup/justfit-backup

# 或者只备份数据库文件
cp ~/.local/share/justfit/justfit.db ~/backup/
```

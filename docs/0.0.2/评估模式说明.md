# 评估模式说明

JustFit 提供四种预设评估模式，帮助您快速分析云平台资源：

## 模式对比

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| **安全模式** | 保守分析，避免误判 | 生产环境初次评估 |
| **节省模式** | 平衡效率与准确性 | 日常资源优化 |
| **激进模式** | 深度分析，发现更多问题 | 测试环境或深度优化 |
| **自定义模式** | 灵活调整各项参数 | 高级用户定制分析 |

## 各模式详细说明

### 安全模式

**配置特点**：
- 分析周期较长（30天）
- 阈值设置严格（CPU 5%、内存 10%）
- 置信度要求高（80%）

**适用场景**：
- 生产环境初次评估
- 不希望遗漏任何资源问题
- 可以接受较高的误报率

**典型输出**：
- 更多僵尸 VM 警报
- 更多规格调整建议
- 较保守的潮汐模式判断

### 节省模式

**配置特点**：
- 分析周期适中（14-21天）
- 阈值设置平衡（CPU 10%、内存 20%）
- 置信度要求适中（60%）

**适用场景**：
- 日常资源优化
- 快速识别明显问题
- 平衡准确性与效率

**典型输出**：
- 适中的僵尸 VM 检测
- 合理的规格调整建议
- 基本的潮汐模式识别

### 激进模式

**配置特点**：
- 分析周期较短（7天）
- 阈值设置宽松（CPU 15%、内存 25%）
- 置信度要求较低（50%）

**适用场景**：
- 测试环境评估
- 快速发现潜在问题
- 可接受较高的漏报率

**典型输出**：
- 较少的僵尸 VM 警报
- 激进的规格调整建议
- 更多的潮汐模式发现

### 自定义模式

**功能特点**：
- 所有参数可手动调整
- 支持微调各项阈值
- 保存为个人配置

**可调整参数**：
- 僵尸 VM 检测：分析天数、CPU/内存阈值、最小置信度
- 规格调整：缓冲比例、P95阈值、调整幅度
- 潮汐检测：分析天数、最小稳定性、变化幅度
- 健康评分：各维度权重配置

## 使用建议

1. **首次使用**：建议选择「安全模式」，全面了解资源状况
2. **日常优化**：使用「节省模式」，快速识别明显问题
3. **深度优化**：切换到「自定义模式」，根据实际情况调整参数
4. **测试验证**：在测试环境使用「激进模式」，验证分析效果

## 参数调整原则

- **CPU/内存阈值**：越高越保守，越低越激进
- **分析天数**：越长越准确，但需要更多历史数据
- **置信度**：越高越严格，避免误报但可能漏报

## 注意事项

- 不同模式可能产生不同的分析结果
- 建议从保守模式开始，逐步调整
- 自定义模式配置可保存供下次使用

# JustFit 0.0.1 Beta 任务清单

> 当前版本聚焦 vCenter 端到端数据采集、分析与报告的可交付能力，目标是把核心流程跑通并验证结果的稳定性。

**版本**: v0.0.1 Beta
**更新时间**: 2026-02-08
**负责人**: JustFit 团队

---

## 1. 版本目标

- 完成 vCenter 数据采集与数据治理链路，保证指标数据可追溯入库。
- 实现僵尸 VM、Right Size、潮汐与平台健康分析并输出可导出的报告。
- 把 Wails 前端 + Go 后端打通，保持任务/报告/连接状态可视化，确保 UI 可用性。

## 2. 当前重点任务

### 2.1 后端与采集
- [x] 搭建 Wails v2 宿主、Go 服务层与 SQLite 存储。
- [x] 实现 vCenter 连接管理、采集任务与指标入库流程。
- [ ] 完善 ETL 验证和断点续采的重试逻辑。
- [ ] 任务调度器状态管理（停止/重试/失败）与日志可检索功能保持稳定。

### 2.2 分析与报告
- [ ] 优化僵尸 VM、Right Size、潮汐、健康评分各项算法的置信度与风险说明。
- [ ] 让分析结果在报告中体现 P95、峰谷信息以及建议动作，保持 UI 展示一致。
- [ ] 完善报告导出（PDF/HTML/JSON）与模板，并确保任务级/实体级筛选可用。

### 2.3 前端与体验
- [x] 主页、连接、任务与分析页面完成基础布局与数据绑定。
- [ ] 补全任务中心轮询、进度、操作按钮（停止、重试、删除）交互。
- [ ] 保证 Collection、Dashboard、TaskDetail 等关键视图在高数据量下响应正常。

## 3. 测试与质量
- [ ] 单元测试覆盖核心模块（采集、算法、任务、报告）并稳定 CI。
- [ ] 集成测试覆盖 vCenter 流程，并验证数据库结构与指标一致性。
- [ ] 性能目标：单 VM 分析 < 2 秒、100 VM 采集 < 5 分钟、报告导出 < 30 秒。

## 4. 后续版本预研（v0.1+）
- H3C UIS 与更多平台接入，但在当前版本文档中仅保留概念层面，研发推进后再拆成新页面。
- 自动化执行、用户管理、告警通知等为 Beta 之后的扩展，当前清单只做标签提醒。
- 重新评估国际化、可观测性与多平台打包策略，作为后续里程碑的一部分。

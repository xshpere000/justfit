## 1. v0.0.1 Beta 发布计划

### 1.1 目标
- 验证 vCenter 采集到分析的链路，确保数据可追溯、报告可导出。
- 让任务中心/连接管理/分析视图在 Wails 前端中可用并对接后端数据。
- 通过测试与监控确保后台采集、分析算法与报告导出稳定。

### 1.2 时间线

| 里程碑 | 时间 | 状态 |
|--------|------|------|
| 架构与数据库评审 | 2026-02-14 | ✅ 已完成 |
| 核心功能联调 | 2026-02-28 | 🔄 进行中 |
| Beta 预发布 | 2026-03-15 | ⏳ 规划 |

### 1.3 功能清单（Beta）

| 模块 | 内容 | 状态 |
|------|------|------|
| 连接管理 | vCenter 连接管理、采集任务创建、测试与状态展示 | 🟢 交付中 |
| 数据治理 | ETL 清洗归一、SQLite 存储、指标与任务仓储 | 🟡 校验中 |
| 分析算法 | 僵尸 VM、Right Size、潮汐、平台健康分析 | 🟡 风险微调 |
| 报告与任务 | 报告生成/导出、任务调度、日志、重试 | 🟡 确认用户体验 |

### 1.4 关键风险
- 采集数据在 SQLite 中的入库一致性，需要完善 ETL 日志与重试。
- 分析算法置信度与界面展示需要联调，避免“建议”与实际数据不对齐。
- Wails 前端在高数据量下的表格性能、任务轮询频率需要压力测试。

## 2. 后续版本（v0.1.0 及以后）

### 2.1 方向
- H3C UIS、更多平台的连接与采集在 v0.1 之后推进，当前文档仅保留概念预览。
- 引入自动化执行、告警推送、用户管理、REST API 等企业级能力。
- 加强国际化、可观测性、包管理与多平台打包部署方案。

### 2.2 评估里程碑
- v0.1.0 正式版：完成多平台支持、用户管理与功能完整性。
- v0.2.0 增强版：增加自动化执行、容量规划、成本分析与企业特性。

## 3. 沟通与反馈
- 每日站会同步 Beta 当前障碍。
- 关键阶段召开评审会（架构/联调/发布）。
- 版本发布后整理用户反馈与缺陷列表，作为下一个版本的 backlog。
